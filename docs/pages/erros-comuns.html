Solução de problemas

Erros comuns do Git e como resolvê-los

Entenda por que determinados erros acontecem, quais comandos ajudam no diagnóstico e como corrigir cada situação sem colocar seu trabalho em risco.

🛡️

Regra de segurança

Antes de executar comandos como reset --hard, clean ou exclusão forçada de branches, verifique o estado do repositório e confirme que nenhuma alteração importante será perdida.

Diagnóstico inicial

Três comandos para executar primeiro

Antes de tentar corrigir o problema, descubra o estado atual do repositório.

01

Verifique o estado

Mostra arquivos alterados, conflitos, branch atual e ações pendentes.

git status
02

Confira a branch

Confirma em qual branch você está trabalhando.

git branch --show-current
03

Consulte o histórico

Exibe os commits recentes e ajuda a localizar alterações importantes.

git log --oneline --graph --all -10
Problemas frequentes

Mensagens de erro e soluções

Clique em cada situação para visualizar a explicação e os passos de correção.

📁 Fatal: not a git repository O terminal não está dentro de um repositório Git.

fatal: not a git repository (or any of the parent directories): .git

Por que acontece?

O comando foi executado em uma pasta que não contém o diretório oculto .git.

Como corrigir

  1. Confira a pasta atual:
    pwd
  2. Liste os arquivos e pastas:
    ls -la
  3. Entre na pasta correta:
    cd caminho/do/projeto
  4. Caso seja um projeto novo, inicialize o Git:
    git init
📝 Nothing added to commit Existem arquivos modificados ou não rastreados, mas eles ainda não foram preparados.

no changes added to commit (use "git add" and/or "git commit -a")

Por que acontece?

O comando git commit foi executado sem que os arquivos fossem adicionados à área de preparação.

Como corrigir

git status git add nome-do-arquivo git commit -m "tipo: descreve a alteração"

Adicionar arquivos individualmente reduz o risco de incluir alterações não relacionadas no commit.

🌿 Src refspec main does not match any A branch informada pode não existir ou ainda não possui commits.

error: src refspec main does not match any

Possíveis causas

  • O repositório ainda não possui nenhum commit.
  • A branch atual possui outro nome.
  • O nome da branch foi digitado incorretamente.

Verifique a branch atual

git branch --show-current

Caso ainda não exista um commit

git add . git commit -m "chore: cria estrutura inicial" git push -u origin main

Substitua main pelo nome realmente apresentado em git branch --show-current.

🔗 Remote origin already exists Já existe um repositório remoto chamado origin.

error: remote origin already exists

Confira o remoto configurado

git remote -v

Caso a URL esteja errada

git remote set-url origin https://github.com/usuario/repositorio.git

Confira novamente

git remote -v

Não é necessário criar outro origin. Atualize a URL do remoto existente.

⬆️ Push rejected — non-fast-forward O repositório remoto possui commits que ainda não estão na sua branch local.

rejected: non-fast-forward

Por que acontece?

Outra alteração foi enviada ao repositório remoto antes do seu push. O Git impede que você sobrescreva esse histórico acidentalmente.

Solução recomendada

git status git pull --rebase origin main git push

Caso surja um conflito durante o rebase, resolva os arquivos e execute:

git add nome-do-arquivo git rebase --continue

Para cancelar o rebase:

git rebase --abort

Evite utilizar git push --force sem compreender o impacto, especialmente em branches compartilhadas.

⚔️ Conflito de merge Duas versões modificaram a mesma região de um arquivo.

CONFLICT (content): Merge conflict in arquivo

Identifique os arquivos em conflito

git status

Localize os marcadores no arquivo 

<<<<<<< HEAD
conteúdo da sua branch
=======
conteúdo da outra branch
>>>>>>> nome-da-branch
  1. Escolha ou combine o conteúdo correto.
  2. Remova os marcadores <<<<<<<, ======= e >>>>>>>.
  3. Salve o arquivo.
  4. Marque o conflito como resolvido:
    git add nome-do-arquivo

Se o conflito ocorreu durante um merge:

git commit

Se ocorreu durante um rebase:

git rebase --continue

Para cancelar um merge ainda não concluído:

git merge --abort
🔄 Please commit or stash your changes Existem alterações locais que impedem a troca de branch ou atualização.

Please commit your changes or stash them before you switch branches

Opção 1 — Criar um commit

git add . git commit -m "wip: salva trabalho em andamento"

Opção 2 — Guardar temporariamente

git stash push -m "trabalho em andamento" git switch nome-da-branch

Para recuperar depois:

git stash list git stash pop

Use o stash para alterações temporárias. Um trabalho importante e coerente deve ser registrado em commit.

🧭 Detached HEAD O Git está apontando diretamente para um commit, e não para uma branch.

You are in 'detached HEAD' state

Caso você não tenha criado alterações

git switch main

Caso tenha criado commits que deseja preservar

git switch -c recovery/trabalho-recuperado

Criar uma branch nesse ponto preserva os commits realizados enquanto o HEAD estava desconectado.

↩️ Arquivo adicionado por engano O arquivo entrou na área de preparação, mas ainda não deveria fazer parte do commit.

Retire o arquivo da área de preparação

git restore --staged nome-do-arquivo

As alterações continuarão no arquivo. Apenas a preparação será desfeita.

Esse comando é diferente de git restore nome-do-arquivo, que descarta alterações locais.

✏️ Mensagem incorreta no último commit O commit foi criado, mas sua mensagem precisa ser corrigida.

Caso o commit ainda não tenha sido enviado

git commit --amend -m "docs: corrige mensagem do commit"

Evite alterar commits que já foram enviados para uma branch compartilhada. O amend muda a identificação do commit.

🔐 Arquivo sensível enviado ao repositório Um arquivo como .env ou uma chave foi incluído por engano.

Remova o arquivo do rastreamento

git rm --cached .env

Adicione ao arquivo .gitignore

echo ".env" >> .gitignore git add .gitignore git commit -m "security: remove arquivo de ambiente"

Caso uma senha, token ou chave já tenha sido enviada ao GitHub, considere a credencial comprometida e substitua-a imediatamente. Remover o arquivo em um novo commit não apaga automaticamente o conteúdo do histórico anterior.

Recuperação

Localizar commits aparentemente perdidos

O reflog registra movimentações recentes do HEAD e pode ajudar na recuperação de commits.

Consulte o reflog

git reflog

Crie uma branch a partir do commit encontrado

git switch -c recovery/commit-recuperado hash-do-commit

Criar uma branch de recuperação é mais seguro do que mover imediatamente a branch principal.

Prevenção

Hábitos que evitam a maioria dos problemas

✅

Use git status

Consulte o estado antes de adicionar arquivos, trocar de branch ou criar commits.

✅

Faça commits pequenos

Um commit deve representar uma alteração coerente e fácil de revisar.

✅

Atualize antes de desenvolver

Receba as alterações da branch principal antes de iniciar uma nova funcionalidade.

✅

Revise o diff

Confira exatamente o que será incluído no commit antes de registrá-lo.

✅

Configure o .gitignore

Exclua arquivos de ambiente, dependências, credenciais e arquivos gerados.

✅

Evite comandos por impulso

Entenda o efeito de comandos destrutivos antes de executá-los.
