Exercício 01

Inicializar um projeto e criar o primeiro commit

Cenário:

Você criou uma pasta chamada meu-primeiro-projeto com um arquivo README.md. A pasta ainda não utiliza Git.

Sua tarefa

  1. Inicialize o repositório.
  2. Confira o estado dos arquivos.
  3. Adicione o README à área de preparação.
  4. Crie o primeiro commit.
Revelar resposta
git init git status git add README.md git commit -m "docs: cria documentacao inicial"

O git init cria o repositório local. O arquivo precisa passar pela área de preparação antes de ser registrado em um commit.

Exercício 02

Criar uma branch de funcionalidade

Cenário:

Você está na branch main e precisa desenvolver um campo de pesquisa sem alterar diretamente a branch principal.

Sua tarefa

Crie e acesse uma branch chamada feature/campo-de-pesquisa. Depois, confirme qual branch está ativa.

Revelar resposta
git switch -c feature/campo-de-pesquisa git branch --show-current

A opção -c cria a nova branch e muda imediatamente para ela.

Exercício 03

Clonar e inspecionar um repositório

Cenário:

Um projeto já existe no GitHub e você precisa baixá-lo para seu computador e verificar a URL remota configurada.

Sua tarefa

Clone o repositório, entre na pasta criada e liste os remotos.

Revelar resposta
git clone https://github.com/usuario/projeto.git cd projeto git remote -v

Normalmente, o Git configura o repositório original com o nome origin.

Nível intermediário

Organização e integração de alterações

Trabalhe com preparação seletiva, stash, merge e limpeza de branches.

Exercício 04

Preparar apenas arquivos relacionados

Cenário:

Os arquivos index.html, style.css e anotacoes.txt foram modificados. Somente os dois primeiros pertencem à funcionalidade.

Sua tarefa

Prepare apenas os arquivos da funcionalidade, revise o conteúdo preparado e crie o commit.

Revelar resposta
git add index.html git add style.css git diff --staged git commit -m "feat: adiciona estrutura visual da pagina"

O arquivo anotacoes.txt continuará modificado, mas não fará parte desse commit.

Exercício 05

Guardar trabalho antes de trocar de branch

Cenário:

Você está desenvolvendo uma funcionalidade ainda incompleta, mas precisa mudar temporariamente para a branch main.

Sua tarefa

Guarde as alterações com uma descrição, troque de branch e depois recupere o trabalho.

Revelar resposta
git stash push -m "campo de pesquisa em desenvolvimento" git switch main git switch feature/campo-de-pesquisa git stash list git stash pop

O stash pop reaplica as alterações e remove a entrada da lista quando a operação termina corretamente.

Exercício 06

Integrar e remover uma branch concluída

Cenário:

A branch feature/menu-mobile foi concluída e deve ser integrada à main.

Sua tarefa

Atualize a branch principal, integre a funcionalidade e remova a branch local concluída.

Revelar resposta
git switch main git pull origin main git merge feature/menu-mobile git branch -d feature/menu-mobile

A opção -d remove a branch apenas quando seu trabalho já foi integrado com segurança.

Nível avançado

Conflitos, reversão e recuperação

Resolva situações que exigem maior atenção ao histórico do repositório.

Exercício 07

Resolver um conflito de merge

Cenário:

Durante um merge, o arquivo README.md apresentou conflito. Você já editou o arquivo e removeu os marcadores.

Sua tarefa

Confirme os conflitos, marque o arquivo como resolvido e finalize o merge.

Revelar resposta
git status git add README.md git commit

Antes de executar git add, confirme que os marcadores do conflito foram removidos e que o conteúdo final está correto.

Para cancelar um merge ainda não concluído, utilize git merge --abort.

Exercício 08

Desfazer um commit já compartilhado

Cenário:

O commit a1b2c3d introduziu um erro e já foi enviado para uma branch utilizada por outras pessoas.

Sua tarefa

Desfaça o commit sem reescrever o histórico compartilhado.

Revelar resposta
git revert a1b2c3d git push

O git revert cria um novo commit que desfaz as alterações do commit informado.

Essa abordagem é mais segura que git reset em branches compartilhadas.

Exercício 09

Recuperar um commit aparentemente perdido

Cenário:

Após uma alteração de branch ou reset, um commit importante não aparece mais no histórico normal.

Sua tarefa

Localize o commit e crie uma branch de recuperação a partir dele.

Revelar resposta
git reflog git switch -c recovery/commit-recuperado hash-do-commit

O reflog apresenta movimentações recentes do HEAD. Depois de identificar o hash, criar uma branch preserva o commit recuperado.

Projeto sugerido: página de perfil pessoal

  1. Crie uma pasta e inicialize o Git.
  2. Adicione index.html, style.css e README.md.
  3. Faça um commit inicial.
  4. Crie uma branch para o cabeçalho.
  5. Desenvolva e registre a alteração.
  6. Integre a branch à main.
  7. Crie um repositório no GitHub.
  8. Configure o remoto e envie a branch principal.
  9. Confira o histórico com git log --oneline --graph --all.
