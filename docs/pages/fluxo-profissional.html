Boas práticas

Fluxo de aprendizado com Git

Aprenda uma sequência segura para desenvolver funcionalidades, registrar alterações e colaborar com outras pessoas em um projeto.

💡

Antes de começar

Não desenvolva novas funcionalidades diretamente na branch principal. Crie uma branch específica para cada tarefa, correção ou funcionalidade.

Processo completo

Da atualização da main ao Pull Request

01

Acesse a branch principal

Primeiro, confirme que você está na branch principal do projeto.

git switch main
02

Atualize o repositório local

Receba as alterações mais recentes antes de começar uma nova tarefa.

git pull origin main
03

Crie uma branch específica

O nome da branch deve indicar claramente o objetivo do trabalho.

git switch -c feature/campo-de-pesquisa
Outros exemplos: fix/corrige-filtro docs/atualiza-readme refactor/organiza-javascript
04

Desenvolva e revise as alterações

Durante o desenvolvimento, consulte o estado do repositório e revise as mudanças realizadas.

git status git diff
05

Adicione os arquivos à preparação

Você pode adicionar todos os arquivos ou selecionar apenas aqueles que pertencem ao commit atual.

git add docs/index.html git add docs/assets/css/style.css

Em projetos profissionais, selecionar os arquivos individualmente ajuda a evitar commits com alterações não relacionadas.

06

Revise o conteúdo preparado

Antes do commit, verifique quais arquivos estão na área de preparação.

git status git diff --staged
07

Crie um commit semântico

A mensagem deve informar o tipo e o objetivo da alteração.

git commit -m "feat: adiciona campo de pesquisa"
08

Envie a branch ao GitHub

No primeiro envio, associe a branch local à branch remota utilizando a opção -u.

git push -u origin feature/campo-de-pesquisa

Nos próximos envios dessa mesma branch, será suficiente executar:

git push
09

Abra um Pull Request

No GitHub, abra um Pull Request da branch de desenvolvimento para a branch main.

  • Escreva um título objetivo.
  • Explique o que foi alterado.
  • Informe como testar.
  • Adicione imagens quando houver mudança visual.
  • Revise a aba de arquivos modificados.
10

Atualize e limpe o ambiente local

Depois que o Pull Request for aprovado e integrado, atualize a branch principal e remova a branch antiga.

git switch main git pull origin main git branch -d feature/campo-de-pesquisa git fetch --prune
Conventional Commits

Tipos de commits mais utilizados

O prefixo ajuda a identificar rapidamente a natureza da alteração.

feat

Nova funcionalidade

Utilizado quando uma capacidade nova é adicionada ao projeto.

feat: adiciona busca por comandos
fix

Correção de problema

Utilizado para corrigir um erro ou comportamento inesperado.

fix: corrige filtro de categorias
docs

Documentação

Utilizado para README, guias, exemplos e textos documentais.

docs: atualiza instruções de instalação
style

Formatação visual

Utilizado para estilos, formatação ou alterações que não modificam a regra de negócio.

style: ajusta espaçamento dos cartões
refactor

Refatoração

Reorganiza o código sem adicionar funcionalidade ou corrigir erro diretamente.

refactor: separa lógica de tema
test

Testes

Utilizado para criação, correção ou atualização de testes automatizados.

test: adiciona testes dos filtros
chore

Manutenção

Representa configurações, dependências e tarefas internas do projeto.

chore: adiciona licença MIT
